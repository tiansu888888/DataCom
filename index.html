Week 3: Data Communications & Networking 第 3 周：数据通信与网络

Core Concepts 核心概念

1. Data Communications & Networking in Business

Data communication is the exchange of data between devices via transmission media (e.g., cables, fiber, wireless). In today's business application systems, networking allows centralization of data, resource sharing (printers, databases), and supports enterprise apps like ERP and CRM. Key components include senders, receivers, messages, transmission media, and protocols (e.g., TCP/IP).

2. Convergence & Unified Communications

Convergence refers to the merging of previously distinct technologies (voice, data, video) into a single shared network infrastructure (IP-based). Unified Communications (UC) takes this further by integrating enterprise communication tools — such as instant messaging, VoIP, video conferencing, and email — into a single seamless user interface, improving business agility and reducing operational costs.

1. 商业中的数据通信与网络

数据通信是设备之间通过传输介质（如电缆、光纤、无线）进行数据交换的过程。在当今的商业应用系统中，网络允许数据集中化、资源共享（打印机、数据库），并支持 ERP 和 CRM 等企业应用。关键组件包括发送方、接收方、消息、传输介质和协议（如 TCP/IP）。

2. 融合与统一通信

融合是指将以前独立的技术（语音、数据、视频）合并到一个共享的网络基础设施（基于 IP）中。统一通信 (UC) 进一步将企业通信工具（如即时通讯、VoIP、视频会议和电子邮件）集成到一个无缝的用户界面中，从而提高业务敏捷性并降低运营成本。

Tips 提示

  • Wireshark: Always select the correct Network Interface Card (NIC) before capturing. Use display filters (e.g., arp or icmp) to reduce noise.
  • Packet Tracer: Ensure you use the correct cable type! Straight-through cables connect different devices (PC to Switch), while crossover cables connect similar devices (PC to PC).
  • Wireshark：在捕获之前始终选择正确的网络接口卡 (NIC)。使用显示过滤器（例如 arp 或 icmp）来减少噪音。
  • Packet Tracer：确保使用正确的电缆类型！直通线连接不同设备（PC 到交换机），交叉线连接相似设备（PC 到 PC）。

Practical Exercises (R07) 实践练习 (R07)

Exercise 1: Cisco Packet Tracer - Two-Node Connection 练习 1：Cisco Packet Tracer - 双节点连接

Task: Open Packet Tracer. Drag two PCs (PC0 and PC1) and a Switch to the workspace. Connect them using Copper Straight-Through cables. Assign IP addresses (192.168.1.10 and 192.168.1.11) to the PCs. Ping from PC0 to PC1 to verify connectivity.

任务：打开 Packet Tracer。将两台 PC（PC0 和 PC1）和一台交换机拖到工作区。使用铜质直通线连接它们。为 PC 分配 IP 地址（192.168.1.10 和 192.168.1.11）。从 PC0 ping PC1 以验证连通性。

Exercise 2: Wireshark (Simple) - Capture Basic ICMP Frames 练习 2：Wireshark（简单）- 捕获基本 ICMP 帧

Task: Start a Wireshark capture on your active interface. Open a command prompt and ping a local device or website (e.g., ping 8.8.8.8). In Wireshark, filter by icmp to observe the request and reply frames.

任务：在活动接口上启动 Wireshark 捕获。打开命令提示符并 ping 本地设备或网站（例如 ping 8.8.8.8）。在 Wireshark 中使用 icmp 过滤，观察请求和回复帧。

Exercise 3: Wireshark (Intermediate) - Observe HTTP & Ethernet Frames 练习 3：Wireshark（中等）- 观察 HTTP 和以太网帧

Task: While capturing, open a web browser and visit a non-secure (HTTP) website if possible, or just filter for http. Stop the capture. Click on an HTTP GET packet. In the packet details pane, expand "Ethernet II" to observe the Source and Destination MAC addresses, and expand "Internet Protocol Version 4" to see the IP addresses.

任务：在捕获时，打开浏览器并尽可能访问一个非安全（HTTP）网站，或仅使用 http 过滤。停止捕获。点击 HTTP GET 数据包。在数据包详细信息窗格中，展开“Ethernet II”以观察源和目标 MAC 地址，并展开“Internet Protocol Version 4”查看 IP 地址。

Exercise 4: Wireshark (Critical) - Convert Raw Code/Encryption to Text 练习 4：Wireshark（关键）- 将原始代码/加密数据转换为文本

Task: Sometimes data payloads look like encrypted code or raw hex (e.g., 55 73 65 72 6e 61 6d 65). Capture a login attempt on an HTTP site. Right-click the HTTP POST packet and select Follow → TCP Stream. Wireshark converts the raw hex code into readable ASCII text, revealing credentials (e.g., user=admin&pass=secret123). This demonstrates why HTTPS encryption is critical!

任务：有时数据有效载荷看起来像加密代码或原始十六进制（例如 55 73 65 72 6e 61 6d 65）。在 HTTP 网站上捕获登录尝试。右键单击 HTTP POST 数据包，选择 Follow → TCP Stream。Wireshark 会将原始十六进制代码转换为可读的 ASCII 文本，暴露凭据（例如 user=admin&pass=secret123。这说明了为什么 HTTPS 加密至关重要！

[Raw Hex Payload captured in Wireshark]: 55 73 65 72 6e 61 6d 65 3d 61 64 6d 69 6e 26 70 61 73 73 77 6f 72 64 3d 73 65 63 72 65 74 31 32 33 [Converted to Text via Follow TCP Stream]: Username=admin&password=secret123 WARNING: Sending data over HTTP exposes raw text. HTTPS (TLS/SSL) encrypts this payload so it cannot be read.
